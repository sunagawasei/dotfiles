# セッションの収穫（学びの還元と総点検）

セッションを閉じる前に実行し、Claude Code運用の効率化資産（skill/rule/メモリ/CLAUDE.md）を更新する。

## ワークフロー

### 1. セッションの振り返り

- このセッションでの (a)ユーザーの訂正・ダメ出し（特に2回以上繰り返されたもの） (b)毎回説明し直した前提知識 (c)繰り返した多段手順 (d)新しく確立した運用・好み (e)非自明な技術的発見（ハマりどころ・制約） (f)返答の形式・トーン・構成についてユーザーが述べた個人的な好み（例: 結論を先に／簡潔に／見出しを使わない／確認の頻度・口調等。コード規約ではなく応答スタイルそのものへの指摘） を洗い出す

### 2. 還元先の分類（公式の使い分け基準）

- **skill**: 繰り返した多段手順・30行超のワークフロー（/skill-nameで呼ぶもの）。**スコープ判断**: そのプロジェクト固有の手順は`<project>/.claude/skills/`（プロジェクトローカル）、複数プロジェクトで再利用する汎用手順は`claude/skills/`（グローバル）
- **rule**（.claude/rules/ or claude/rules/）: 特定ファイル種別・作業種別への一貫した規約（pathsで自動適用）。同様にプロジェクト固有かグローバル共通かで置き場所を分ける
- **メモリ**: feedback（ユーザーの指示・訂正+Why/How） / project（進行状況・非自明な制約） / reference（外部情報・環境事実） / user（好み・背景）。1事実1ファイル+MEMORY.mdインデックス。**スコープ判断**: auto memoryは作業ディレクトリ（プロジェクト）ごとに保存領域が分かれる。プロジェクト固有の実装・業務に閉じる学びはそのプロジェクトで作業中のセッションのメモリに置くのを基本とする（本セッション=`~/.config`にはClaude Code運用や複数プロジェクト共通の知見を保存する）
- **CLAUDE.md**: 毎セッション必要な事実・規約のみ（グローバル=全プロジェクト共通の好み、プロジェクト=そのリポジトリの前提）。200行制限に注意
- **hook/設定（settings.json）**: 「毎回確実に」実行/禁止したいもの、環境変数、パーミッション設定（CLAUDE.mdは強制ではない）。グローバル（`~/.claude/settings.json`）かプロジェクトローカル（`<project>/.claude/settings.json`）かは適用範囲で判断（詳細は`update-config` skill参照）
- **グローバルへの昇格チェック（プロジェクトローカルとグローバルは排他ではない）**: 候補をプロジェクトローカルに分類した場合でも、そこから複数プロジェクトに敷衍できる一般則（Claude Code運用そのもの・ツールの癖・環境制約等）を抽出できるなら、その汎用部分だけを抽象化してグローバルメモリまたはグローバル`claude/CLAUDE.md`にも合わせて反映すべきか検討する。プロジェクト固有の実装詳細（固有名詞・具体的な設定値等）はプロジェクト側に残し、汎用部分だけを切り出す
- **(f)の返答スタイルの好みは既定でグローバル扱い**: プロジェクト固有の作業内容ではなく応答の作り方そのものなので、原則グローバル（`claude/CLAUDE.md`の出力規約・対話規約 or グローバルfeedbackメモリ）に振り分ける。プロジェクト限定と明言された場合のみそのプロジェクトのCLAUDE.md/メモリに留める

### 3. 既存資産との突き合わせ

- 候補ごとに既存のskill/rule/メモリ/CLAUDE.mdをgrepし、既にあれば「更新」、なければ「新規」に振り分け。重複を作らない
- スコープ判断を伴う候補（skill/rule/メモリ/設定）は、対象プロジェクト側とグローバル（`~/.config`）側の両方を確認し、同じ内容を両スコープに重複して置かない
- (f)の候補は特にグローバルCLAUDE.mdの「出力規約」「対話・確認の規約」セクションと突き合わせ、既存文言の微調整で吸収できないか優先確認する

### 4. ユーザー確認

- 候補一覧（還元先・スコープ〔プロジェクトローカル/グローバル〕・新規/更新・一行要約）をAskUserQuestionで提示し採否を取る

### 5. 適用

- 承認分をsonnetサブエージェント（model: sonnet）に委譲して適用（メインは検証）。dotfilesの変更はユーザー承認後にcommitエージェントでコミット

### 6. やり残し総点検

- セッション内の「あとで対応」「必要なら実施します」等の約束・提案・保留、サブエージェント報告の「未対応の提案」「スコープ外」項目を洗い出し、未実施リストを提示して対応要否を確認

## 関連

- 常時規約: 同チェックはタスク完了時にも都度行う（グローバルCLAUDE.md「対話・確認の規約」）
- 定期棚卸し（資産全体の重複・矛盾・陳腐化）は `/claude-audit`（こちらは蓄積後の掃除、session-harvestは都度の還元）
