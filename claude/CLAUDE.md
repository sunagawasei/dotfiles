# Golang Guidelines

## Version

- Always use the latest stable version of Go , best practices, and Go idioms.
- go の standard library を優先して使用してください

## Code Design

- テスト容易性を高めるために Dependency Injection (DI) を用いた実装を行う
- サードパーティライブラリは使用しない

## Testing

- テストは明示的に指示された場合のみ生成する
- `t.Run()`を用いたサブテストの形でテストケースを列挙する
- テーブルテストは基本的には用いず

# Display Preferences

## CLI記号による表示設定（強制事項）

**以下のルールは絶対に遵守してください**：

- 絵文字は一切使用禁止（検出次第エラーとする）
- Nerd Fontsアイコングリフのみを使用（例外なし）
- すべての出力でNerd Fontsグリフを必須使用
- Unicode記号（✓✗など）も使用禁止
- フォールバック時もテキスト表示 [OK] [NG] を使用（絵文字化禁止）

## 出力時のチェックリスト

1. 絵文字が含まれていないか確認
2. Nerd Fontsアイコンが正しく使用されているか確認
3. ステータス表示が指定のグリフか確認

## 実装確認（Claude Code動作検証用）

出力前に以下を必ず確認：

- [ ] 絵文字ゼロ（一つでも含まれていたら修正）
- [ ] Nerd Fontsアイコン使用（該当箇所すべて）
- [ ] マッピング準拠（指定されたコードポイント使用）

違反例：

- ❌ "✅ タスク完了" → 違反（絵文字使用）
- ✅ " タスク完了" → 正しい（Nerd Fonts使用）

### 最終チェック項目

1. **すべてのステータス表示**が などのNerd Fontsグリフになっているか
2. **絵文字が一切含まれていない**か（✅❌📊📝🚀🔥💡⭐など）
3. **Unicode記号（✓✗◉○など）を使用していない**か
4. **マッピング表に従った正しいコードポイント**を使用しているか
